# WeekFit Coach Contract Audit

## Required Visible Contract

Every visible Coach state must render all four fields before it appears:

* Assessment: what is happening.
* Recommendation: what to do in the next 30 minutes.
* Warning: what to avoid.
* Confidence: why Coach is confident or limited.

If any field is unavailable, the state should render a bounded missing-data Coach state, not a partial Coach card.

## Contract Gap Summary

The canonical `CoachFinalStory` model contains most of the ingredients: title, subtitle, primary recommendation, avoid recommendation, what happened, what matters, reasons, confidence, and data readiness. The visible UI does not currently enforce that all four required fields are shown. The biggest missing field is visible confidence. The second biggest gap is Today, where the card can show only badge/title/subtitle without the recommendation, warning, or confidence.

## Implementation Contract

Introduce a single visible contract model:

```swift
struct CoachVisibleContract {
    let stateID: String
    let title: String
    let assessment: String
    let recommendation: String
    let warning: String
    let confidence: String
}
```

Rules:

* `assessment`, `recommendation`, `warning`, and `confidence` must be non-empty after trimming.
* `recommendation` must include a concrete next-30-minute action or explicitly say no action is needed and why.
* `warning` must name a concrete mistake to avoid.
* `confidence` must include either a supporting signal or a missing/limited signal.
* Today and Coach must render from the same contract model.
* Support sections, reasons, and bullets may only render after the contract passes.

## Definition Of Done

Every visible Coach state satisfies:

* The user can say what is happening.
* The user knows what to do in the next 30 minutes.
* The user knows what mistake to avoid.
* The user knows why Coach is confident or limited.

No visible Coach state should render if it cannot answer all four.
